# Questioner Database Schema & Architecture

This document describes the necessary database tables (entities) required to persist data for the working flow of the interview platform (excluding user management like auth/login).

## 1. InterviewSession
The core entity linking verification, questions, and evaluation.
* **id** (`UUID`, Primary Key): Unique id for the session.
* **candidate_name** (`String`): Candidate's recognized or parsed name.
* **face_verified** (`Boolean`): Whether the candidate passed face verification.
* **origin_url** (`String`, optional): The Vercel app URL that triggered the session.
* **created_at** (`Timestamp`): Time the session was generated.
* **submitted** (`Boolean`): Flag indicating if evaluations have already run (prevents duplicate submission).
* **submitted_at** (`Timestamp`, optional): Time the evaluations were submitted.

## 2. CandidateProfile
Stores the extracted knowledge from a candidate's resume/audio to inform the LLM context.
* **session_id** (`UUID`, Foreign Key): Links to InterviewSession.
* **skills** (`Text`): Parsed skills.
* **projects** (`Text`): Candidate projects.
* **experience** (`Text`): Work experience.
* **education** (`Text`): Educational background.
* **certifications** (`Text`): Candidate certifications.
* **speech_transcript** (`Text`): Audio self-introduction transcript.
* **additional_information** (`JSON`): Any other supplementary resume data.

## 3. Question
The list of AI-generated questions specific to a session. 
* **id** (`String/UUID`, Primary Key): Uniquely identifies the question.
* **session_id** (`UUID`, Foreign Key): Links to InterviewSession.
* **ordinal_number** (`Integer`): 1-15 representing its order in the interview.
* **difficulty** (`Enum`): 'easy', 'medium', or 'hard'.
* **category** (`Enum`): 'technical', 'project', 'behavioural', or 'general'.
* **question_text** (`Text`): The question presented to the candidate.

## 4. IdealAnswer
The reference answer generated by the LLM, kept securely on the backend.
* **question_id** (`String/UUID`, Foreign Key): Links to Question.
* **ideal_answer_text** (`Text`): The concise, correct reference answer used by the evaluation LLM.

## 5. AnswerEvaluation (Question Result)
The candidate's answer and its leniency-scored evaluation.
* **session_id** (`UUID`, Foreign Key): Links to InterviewSession.
* **question_id** (`String/UUID`, Foreign Key): Links to Question.
* **candidate_answer** (`Text`): The response submitted by the candidate.
* **score** (`Integer`): 0–10 score assigned by the evaluating LLM.
* **feedback** (`Text`): Short constructive feedback provided by the evaluating LLM.
* **badge** (`Enum`): qualitative assessment (e.g., 'Excellent', 'Good', 'No Attempt').

## 6. InterviewResult
The aggregated performance data for the candidate's interview session.
* **session_id** (`UUID`, Foreign Key): Links to InterviewSession.
* **total_score** (`Integer`): Summation of all individual answer scores.
* **max_score** (`Integer`): Maximum available score (e.g. number of questions * 10).
* **percentage** (`Float`): The final percentage (0.0 to 100.0).
* **grade** (`Enum`): Qualitative letter grade ('A+', 'A', 'B', 'C', 'D', 'F').
* **summary** (`Text`): A brief AI-generated summary of overall performance.

---

### Project Flow
1. **Verification**: Candidate completes a face verification step, generating a secure token.
2. **Context Setup**: Candidate uploads resume and audio, processed into a unified context profile.
3. **Question Generation**: System passes the profile to LLM to create 15 custom questions and hidden ideal reference answers.
4. **Interview**: Candidate views questions, records their answers in the portal, and submits them.
5. **Evaluation**: System compares candidate submissions to ideal answers returning per-question grades, feedback, and an overall report.
